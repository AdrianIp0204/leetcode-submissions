# Model Answers

These notes are the clean reference answers Adrian can compare against the
actual submitted solutions under `submissions/`.

The rule is simple: preserve the real attempt, then write the model answer here.
Do not rewrite weak early submissions to look stronger.

Each model-answer note should include:

- the intended pattern;
- a compact model solution;
- why that solution is the one to learn from;
- the invariant or proof idea;
- complexity;
- one review prompt.

Run this after adding notes:

```bash
npm run audit:model-answers
```
