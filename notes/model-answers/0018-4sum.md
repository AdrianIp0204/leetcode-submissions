# 4Sum

- Pattern: sorting + fixed indices + two pointers
- Original attempt: [submissions/0018-4sum](../../submissions/0018-4sum/)

## Model Solution

```python
class Solution:
    def fourSum(self, nums: List[int], target: int) -> List[List[int]]:
        nums.sort()
        res = []
        n = len(nums)

        for i in range(n - 3):
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            for j in range(i + 1, n - 2):
                if j > i + 1 and nums[j] == nums[j - 1]:
                    continue

                left = j + 1
                right = n - 1
                need = target - nums[i] - nums[j]

                while left < right:
                    pair = nums[left] + nums[right]
                    if pair == need:
                        res.append([nums[i], nums[j], nums[left], nums[right]])
                        left += 1
                        right -= 1

                        while left < right and nums[left] == nums[left - 1]:
                            left += 1
                        while left < right and nums[right] == nums[right + 1]:
                            right -= 1
                    elif pair < need:
                        left += 1
                    else:
                        right -= 1

        return res
```

## Why This Is The Model

The accepted attempt is already the standard solution. Sorting makes the inner
two-pointer movement monotonic, and duplicate skipping at every chosen position
keeps each quadruplet unique.

## Invariant

For fixed `i` and `j`, every pair outside the current `[left, right]` window has
either been checked or ruled out by sorted order. Moving `left` raises the pair
sum, and moving `right` lowers it.

## Complexity

- Time: O(n^3)
- Space: O(1) extra, ignoring output and sort internals
